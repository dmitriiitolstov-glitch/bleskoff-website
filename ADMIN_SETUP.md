# Подключение админки BleskOFF

Админка сделана на Sanity Studio. Клиент получает отдельный вход и меняет содержимое в браузере, а исходный код остаётся в GitHub.

## 1. Создать проект Sanity

1. Зарегистрируйтесь на [sanity.io](https://www.sanity.io/) через GitHub или почту.
2. Создайте проект `BleskOFF` с набором данных `production`.
3. Скопируйте `Project ID` в настройках проекта.

## 2. Подключить проект локально

Скопируйте `.env.example` в `.env` и вставьте Project ID:

```env
PUBLIC_SITE_URL=https://blesk-off.ru
PUBLIC_SANITY_PROJECT_ID=ваш_project_id
PUBLIC_SANITY_DATASET=production
SANITY_STUDIO_PROJECT_ID=ваш_project_id
SANITY_STUDIO_DATASET=production
```

Затем импортируйте подготовленные записи и запустите сайт:

```sh
pnpm cms:import
pnpm dev
```

Админка откроется по адресу `http://localhost:4321/admin`, сам редактор — `http://localhost:4321/studio/`.

## 3. Дать доступ клиенту

В Sanity откройте `Manage → Members`, пригласите клиента по почте и выберите роль Editor. Пароли друг другу передавать не нужно.

## 4. Подключить опубликованный сайт

В Netlify добавьте переменные из `.env` в `Site configuration → Environment variables`. После этого выполните новую публикацию сайта.

Чтобы изменения из админки автоматически появлялись на сайте:

1. В Netlify создайте `Build hook`.
2. В Sanity создайте webhook на этот адрес для событий Create, Update и Delete.
3. После нажатия `Publish` Sanity запустит новую сборку сайта.

## Что клиент сможет менять

- акции, размеры скидок, сроки и условия;
- услуги и фотографии;
- готовые программы;
- вопросы и ответы;
- главный заголовок, описание и кнопку;
- телефон, адрес, режим работы и ссылки;
- SEO-заголовок, описание и изображение для соцсетей.

Технические файлы, стили и логика сайта остаются в GitHub. Изменения кода лучше делать отдельными коммитами, чтобы любую версию можно было вернуть.
